I will implement a dynamic social media management system for both the Hero section and Footer, backed by a database with icon upload capabilities.

### **Plan:**

1.  **Database Migration**:
    *   Create a new `social_links` table to store individual social media entries.
    *   Columns: `id`, `platform` (name), `url`, `icon` (file path), `is_active` (boolean), `sort_order` (integer).
    *   Remove the hardcoded JSON `social_links` column from `footer_settings` (optional, or just stop using it).

2.  **Model Creation**:
    *   Create `App\Models\SocialLink` model.

3.  **Admin Panel - Social Links Management**:
    *   Create a new Livewire component `App\Livewire\Admin\SocialLinks\Index` (and view) to manage these links (CRUD).
    *   Features: Add new link, Upload Icon, Edit URL/Name, Delete, Toggle Active status.
    *   Add a navigation link in the Admin Sidebar for "Social Links".

4.  **Frontend Integration**:
    *   **Hero Section (`home.blade.php`)**: Update to fetch and display active `SocialLink` records dynamically.
    *   **Footer (`layouts.public.blade.php` or `footer.blade.php`)**: Update to fetch and display the same active `SocialLink` records.

This approach provides a single source of truth for social links, allowing you to easily add new platforms (e.g., TikTok, Threads) with custom icons without touching the code.
